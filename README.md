# CoScribe

CoScribe 是一款以本地文件夹为项目载体的 AI 学习与内容工作台。它把 PDF/DOCX 阅读、Markdown 编辑、本地 OCR、项目搜索、独立 AI 会话和安全的笔记写入放在同一个 Electron 桌面应用中；不需要账号，也不会把用户文件转换为只能由本软件读取的私有格式。

## 下载

前往 [GitHub Releases](https://github.com/UESTC-ly/CoScribe/releases) 下载最新版本。发布产物包括 Apple Silicon macOS 的 DMG/ZIP 和 Windows x64 的 NSIS 安装程序。由于尚未配置 Apple Developer ID 和 Windows 代码签名证书，系统首次打开时可能显示来源或 SmartScreen 提示。

## 已实现能力

- 新建本地项目、打开任意现有文件夹、最近项目与失效路径提示。
- 保留真实目录结构的文件树：创建、重命名、移动、拖入、刷新、移到系统废纸篓和文件管理器定位。
- PDF 连续阅读、虚拟化页面、缩放、适合宽度/页面、页码跳转、缩略图、目录、文内搜索、文本选择、高亮、批注、书签和阅读位置恢复。
- DOCX 本地语义解析、净化后预览、正文复制、项目搜索和 AI 上下文；不会把文档上传到第三方转换服务。
- Markdown 源码编辑、实时预览、GFM、任务列表、表格、代码、数学公式、Mermaid、大纲、查找、撤销/重做、自动/手动保存、相对图片和项目内链接。
- 图片与文本查看；图片和扫描型 PDF 当前页可使用内置 PP-OCRv6 本地识别，模型随安装包提供，无需首次下载。
- 可选 AI 增强 OCR：仅在用户明确点击时，把当前图像或 PDF 页面发送到已配置的 AI 服务，并在界面中标明需对照原文校对。
- 多标签、标签拖动排序、左右分屏、活动区域识别和冷启动现场恢复。
- 文件名、Markdown、文本、DOCX、PDF 文本层/OCR、会话标题和消息的项目搜索，并保留页码/章节/行号。
- 每个项目内的多 AI 会话、独立历史、自动标题、流式回答、停止、重新生成、来源跳转和发送时上下文快照；状态栏可切换模型和思考强度。
- AI 回答支持 GFM、数学公式、Mermaid 图表和常用语言语法高亮；代码块可直接复制，未知语言安全回退为纯文本。
- AI 面板支持 300-800px 动态宽度、拖拽、键盘调整和双击复位；宽屏保留中央编辑空间，窄屏自动切换覆盖布局。
- 五档 AI 上下文：选中内容、当前内容、当前文档、当前项目和通用知识。默认不会扫描整个项目。
- AI 创建、追加或替换 Markdown 前展示完整预览或差异；只有点击“接受并写入”后才落盘。
- 外部 Markdown 修改冲突检测，可选择载入磁盘版本、保留本地编辑或查看并排差异。
- 浅色、深色和跟随系统主题；字体、自动保存、默认上下文和通用知识策略可配置。

## 快速开始

需要 Node.js 20 或更高版本。

```bash
npm install
npm run dev
```

生产构建与本地预览：

```bash
npm run build
npm start
```

生成当前平台的未安装应用目录：

```bash
npm run dist:dir
```

生成 DMG/ZIP、NSIS 或 AppImage/DEB 等分发包：

```bash
npm run dist
```

## 配置 AI

打开“设置 → AI 服务”，填写：

- 服务地址：OpenAI-compatible API 的基础路径，例如 `https://api.openai.com/v1`，也可以填写裸域名或完整接口地址。
- 接口协议：默认“自动”。裸域名优先使用 Responses API；包含 `/v1` 的传统基础路径默认使用 Chat Completions；完整的 `/responses` 或 `/chat/completions` 地址会直接决定协议。也可以手动指定协议。
- 模型：服务端可用的模型名称。
- API Key：远程服务必须填写；本机 `localhost`、`127.0.0.0/8` 或 `::1` 服务可以无 Key 运行。

远程服务只允许 HTTPS，避免 Key 通过明文 HTTP 发送。API Key 使用 Electron `safeStorage` 加密后保存在操作系统用户数据目录，不写入项目。AI 未配置时，所有本地阅读、编辑、搜索和状态恢复能力仍可正常使用。

## 本地数据与安全边界

项目就是用户选择的普通文件夹。PDF、DOCX、Markdown、图片和文本始终是标准磁盘文件，可由其他软件直接访问。

应用仅在项目根目录使用 `.vibeknowledge/` 保存工作区、会话、标注和 OCR 缓存元数据。CoScribe 在更名后继续沿用这一隐藏目录及旧版用户数据路径，以确保现有项目、会话、设置和加密 API Key 无损恢复。写入采用临时文件、同步和原子替换；损坏的 JSON 元数据会回退到安全默认值，不会修改原始内容。

所有渲染进程文件调用都经过主进程校验：

- 拒绝 `..` 越级路径、项目外绝对路径、符号链接跳转和元数据目录写入。
- Markdown 保存会检查磁盘修改时间，外部变更时拒绝静默覆盖。
- AI 只能提出当前项目内 `.md`/`.markdown` 的创建、追加或替换建议，不能删除文件或覆盖二进制文件。
- AI 建议在主进程注册并绑定精确内容；预览后目标发生变化或文件被外部修改时，确认也会失败。
- DOCX 转换结果在渲染前使用 DOMPurify 净化；本地 OCR 在沙箱化 Worker 中运行，内置模型和 WASM 资源只从应用包加载。
- Electron 启用沙箱、上下文隔离、禁用 Node 集成，并校验 IPC 来源。

## 上下文规则

默认“当前内容”模式会优先使用选中文字，否则使用当前 PDF 页、Markdown 章节或可见段落。只有明确选择“当前文档”才读取全文，明确选择“当前项目”才执行项目级相关内容检索；“通用知识”不会附带项目正文。

每次发送会固化项目、会话、活动分屏、文档、页码/章节、选中文字、范围和引用文件。发送后切换标签不会改变该问题的上下文。项目来源由应用根据真实文件生成，模型不能自行声明一个不存在的项目来源。

## 验证

```bash
npm run typecheck     # Renderer、preload 和 main TypeScript
npm test              # 单元、Electron 安全与 AI 组件测试
npm run test:e2e      # 真实 Electron 端到端流程
npm run build         # 生产构建
```

端到端套件会创建隔离的临时项目和用户数据目录，验证真实磁盘搜索/编辑/保存、冷启动恢复、DOCX 阅读、本地 OCR Worker、当前文档笔记路径，以及本地模拟模型驱动的“AI 建议笔记 → 确认前不落盘 → 确认后写入”。

## 目录结构

```text
electron/
  main/       主进程：项目、路径安全、存储、PDF 提取、搜索和 AI
  preload/    受限 IPC API
src/
  components/ 工作区、阅读器、编辑器和 AI 界面
  lib/        上下文、Markdown 章节、路径与恢复逻辑
  store/      Zustand 应用状态
  shared/     主进程与渲染进程共享类型
tests/
  unit/       纯逻辑和状态测试
  e2e/        Playwright Electron 验收流程
```

## 当前边界

本版本不包含登录、云同步、团队协作、PDF 原文编辑、联网搜索、移动端或自动无确认写文件。OCR 结果会保留引擎信息和校对提示；CoScribe 不会把机器识别结果伪装成已核验原文。
